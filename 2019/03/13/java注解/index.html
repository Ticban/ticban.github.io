Java注解

注解Annotation是JDK5开始引入的新特性，Java注解是附加在代码中的一些元数据，所谓元数据即是描述数据的数据，
用于一些工具在编译、运行时进行解析和使用，起到说明、配置的功能。注解不会也不能影响代码的实际逻辑，仅仅起到辅助性的作用。包含在 java.lang.annotation 包中。
如hibernate可以通过注解将类中实体字段和数据库字段实现映射关系，同时可以指定主键。

通过注解，指定实体类Team和表teams呈现映射关系，且id字段为表的主键，表中的name列不能为null。

注解的用处

  1. 生成文档。常用的有@param @return等，是java最早提供的注解
  2. 跟踪代码依赖性，实现替代配置文件功能。
  3. 在编译时进行格式检查，如@override表明这个方法是重写父类的超类方法，则编译时就能检查出。

注解的原理

注解本质上是继承了Annotation的特殊接口，其具体实现类是Java运行时生成的动态代理类。而我们通过反射获取注解时，返回的是Java
运行时的动态代理对象$Proxy1。通过代理对象调用自定义注解的方法，会最终调用AnnotationInvocationHandler的invoke方法。该
方法会从memberValues这个Map中索引出对应的值。而memberValues的来源是Java常量池（即注解中的值存放在常量池中）。

自定义注解需要使用以下的注解：

  • @Documented 表示在生产JavaDoc文档的时候把注解信息也包含进去

  • @Retention 表示该注解的级别。可选的范围有：

    1. SOURCE：只在源代码中出现，编译后就被丢弃了，如@Override,@SuppressWarnings
    2. CLASS:会被编译到class文件里
    3. RUNTIME：在jvm加载class文件的时候，会把注解的信息加载进去，可用于运行时反射，自定义注解时常用的级别。
    • @Target 表示该注解的作用位置
    • @Inherited 表示子类继承父类的注解

    注解仅仅是元数据（为类，方法，字段，包提供信息），它本身并不包含任何的业务逻辑，通常另有一块代码来读取注解中的信息，处理相应的业务逻辑。
