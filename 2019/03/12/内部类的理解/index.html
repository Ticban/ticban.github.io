内部类的理解

发表于 2019-03-12 | 分类于 java基础

内部类

什么是内部类

  • 内部类是指再一个外部类的内部再定义一个类。内部类作为外部类成员的一部分，并且依附于外部类而存在的。
  • 内部类可用private和protect修饰，而外部类只能使用public和缺省的包访问权限
  • 内部类可以为静态的，静态内部类独立于外部类而存在，只能访问外部类的静态属性和方法。
  • 内部类主要有成员内部类、局部内部类、静态内部类、匿名内部类

局部内部类

在方法中定义的内部类称为局部内部类。与局部变量类似，局部内部类也不能有访问修饰符修饰。局部内部类只能在定义该内部类的方法内实例化，不可以在此方法外对其实例化。且只在方法被执行时加载

匿名内部类

匿名内部类是局部内部类的一中特殊形式。没有名字的内部类。
使用内部类有几个原则：

  • 匿名内部类不能有构造方法
  • 匿名内部类不能定义任何静态成员、方法和类
  • 匿名内部类不能是public、private、static、protect
  • 只能创建匿名内部类的一个实例
  • 一个匿名内部类一定是在new的后面，用其隐含实现一个接口或者实现一个类
  • 因匿名内部类为局部内部类，所以局部内部类的所有限制都对其生效

使用匿名内部类的场景：
类在定义后马上用到；类非常小；给类命名并不会导致你的代码容易被理解。

匿名内部类的final

当给匿名内部类传递参数的时候，该形参在内部类中需要被使用。那么该形参必须要为final。

为什么必须要为final

内部类编译后的代码：

匿名内部类并不是直接调用方法传递的参数，而是利用自身的构造器对传入的参数进行备份，自己内部方法调用的其实是自己的属性而不是外部方法传递的参数本身。所以两者是可以任意变化的、相互独立的。
所以使用final来避免形参的改变

匿名内部类的初始化

匿名内部类不能像其它类一样，它不能使用构造函数。但是可以使用构造代码块初始化

内部类能被重载吗

内部类并不能被重载，因为内部类独立于外部类，如果这两个内部类是完全独立的两个实体，按照反编译的结果来看，它们拥有不同的命名空间。

内部类的共性

  1. 内部类仍然是一个独立的类，在编译后内部类会被编译成独立的class文件，但是前面冠以外部类的类名和$符号。
  2. 内部类不能用普通的方式访问。
  3. 外部类不能直接访问内部类的成员，但是能通过内部类对象来访问

内部类是外部类的一个成员，内部类可以自由的访问外部类的成员变量。 因为当某个外部类的对象创建内部类的对象时，此内部类会捕捉一个隐式引用，它引用了实例化该内部对象的外部类对象，通过这个指针，可以访问外部类对象的全部状态。
通过反编译内部类的字节码，主要通过一下几步做到的：

  1. 编译器自动为内部类添加一个成员变量，这个成员变量类型和外部类类型相同。这个成员变量就是外部类实例的引用。
  2. 编译器自动为内部类的构造方法添加一个外部类参数，主要为添加的成员变量赋值。
  3. 在调用内部类的构造函数初始化内部类对象时，会默认传入外部类的引用。

静态内部类(嵌套类 nested class [C++说法])与普通内部类(Inner class)的区别

  • 静态内部类不持有外部类的引用：在普通内部类中，可以直接访问外部类的属性、方法，这是因为内部类持有外部类实例的引用，可以自由访问。而静态内部类只能访问外部类中的静态属性和方法。
  • 静态内部类的创立不依赖外部类：静态内部类和外部类是独立的，即使外部类消亡了，静态内部类还是存在的。而普通内部类的创建依赖于外部类的实例，即先创建外部类的实例，依靠外部类的实例才能创建内部类。生命周期和外部类相同。

使用内部类的好处

静态内部类的使用：

  • 降低包的深度，方便类的使用。静态内部类适用于包含类当中，但又不依赖于外部类。
  • 静态内部类的创建不需要依赖外部类，

非静态内部类：

  • java只支持单继承，内部类的参与可以实现多继承的解决方案
  • 外部类的继承和实现接口与内部类的继承和实现无关，内部类不受其影响。
